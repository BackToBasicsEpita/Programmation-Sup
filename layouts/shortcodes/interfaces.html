Introduction : Qu'est ce qu'une interface


Rapidement (oui oui on vous voit ceux qui n'ont pas relu le cours de GolluM), les Interfaces ressemblent en fait beaucoup à des classes abstraites.
L'idée est d'écrire dans une interface "mère" toutes les fonctions / attributs sans corps (contrairement aux classes abstraites) que nous allons utiliser par la suite.

L'analogie du contrat

En programmation, l'analogie du contrat est souvent utilisée afin d'expliquer le concept d'interface.
Lorsque vous créez une interface, vous spécifiez les méthodes, propriétes et évènements que les classes qui implémentent cette interface doivent fournir.
En d'autres termes, vous définissez un contrat entre l'interface et les classes qui l'utilisent.

Pour comprendre cette analogie, imaginez que vous êtes un entrepreneur (Oussama Amar par exemple) et que vous devez embaucher un travailleur indépendant pour effectuer un travail spécifique pour vous.
Avant de signer un contrat avec le travailleur indépendant, vous devez vous mettre d'accord sur les termes du contrat, tels que le travail à effectuer, le délai de livraison et le paiement.
Vous définissez ainsi un contrat entre vous et le travailleur indépendant.

De même, en C#, une interface définit un contrat entre l'interface et les classes qui l'implémentent.
Le contrat spécifie les méthodes, propriétés et événements que les classes doivent fournir pour respecter l'interface.

Ainsi, lorsqu'une classe implémente une interface, elle s'engage à fournir les fonctionnalités spécifiées par l'interface, tout comme un travailleur indépendant s'engage à fournir les services spécifiés dans un contrat.
Une interface sert aussi à structurer son code, afin de ne pas partir dans tous les sens !

Pour rendre cela plus concret, voilà un exemple qui consistera en l'implémentation d'un mini jeu en tour par tour:

Un super exemple

      public interface IPersonnage
      {
        int PV { get; set; }
        void Attaquer(IPersonnage cible);
      }

      public class Guerrier : IPersonnage
      {
          public int PV { get; set; }
          public int Attaque { get; set; }

          public void Attaquer(IPersonnage cible)
          {
              cible.PV -= Attaque;
          }
      }

IPersonnage


L'interface IPersonnage est utilisée pour définir une abstraction commune pour tous les personnages du jeu de combat.
Elle comprend les membres suivants :
- La propriété PV : cette propriété est utilisée pour stocker le nombre de points de vie d'un personnage.
- La méthode Attaquer(IPersonnage adversaire) : cette méthode est utilisée pour attaquer un adversaire.
Elle prend une référence de type IPersonnage comme paramètre, ce qui permet à n'importe quel type de personnage d'être attaqué.

Cette méthode est implémentée de manière spécifique pour chaque type de personnage, c'est-à-dire que la façon dont un Guerrier attaque sera différente de celle d'un Magicien (par exemple).

Il est donc important de comprendre qu'une Interface n'a aucune fonction avec un corps !

Il faut aussi savoir que la convention de nommage d"une interface est de mettre un I majuscule devant !


Guerrier


La classe Guerrier est une implémentation concrète de l'interface IPersonnage. Elle comprend les membres suivants :

- La propriété PV : cette propriété est héritée de l'interface IPersonnage..
- La propriété Attaque : cette propriété est utilisée pour stocker la force d'attaque du guerrier.
- La méthode Attaquer(IPersonnage adversaire) : cette méthode est implémentée pour le guerrier en fonction de sa spécificité.
Lorsqu'un guerrier attaque un adversaire, il soustrait à ses points de vie le montant de son attaque.

La classe Warrior hérite de l'interface IPersonnage
Elle définit un attribut PV (avec ses getters et setters) et un attribut damages (avec ses getters et setters)
Comme la classe Warrior hérite de IPersonnage, elle définie aussi Attack.
Ici, on retirera juste le nombre de Damages aux PV de la target; et si les PV deviennent négatifs, on les remet à 0

La classe Wizard est très similaire à la classe Warrior. Elle hérite aussi de l'interface IPersonnage

Ici, on retirera juste le nombre de MagicalPower aux PV de la target; et si les PV deviennent négatifs, on les remets à 0

Cette fonction sera appelée dès qu'un personnage attaque un ennemi.
Le fonctionnement est assez simple :

Maintenant que la fonction d'attaque a été codée, il faut bien pouvoir afficher les Teams !

Voilà les étapes à suivre pour afficher une Team :
- On affiche le nom à qui la Team appartient
- Puis, sur la même ligne, on affiche tous les personnages de cette team, séparés par le caractère |

Pour afficher un personnage on respecte la convention suivante :
| ([N°Perso]) [Nomperso] - PV : [PV] |
(Ici chaque mot entre crochets doit être remplacé par une valeur)

Hésitez pas à regarder l'exemple en dessous pour plus de claireté :)

Le numéro correspond à la place du personnage dans la liste (le numéro commence à 1)

Si ce n'est pas clair, regarder l'exemple en dessous.

Ici, nous allons faire en sorte que l'ordinateur choisisse un personnage dans sa Team, et attaque un personnage ennemi (de la Team du joueur)

Avant de commencer la fonction, mettre la couleur de la console en rouge (car c'est une action de l'ordinateur !)
Pour ce faire utiliser : Console.ForegroundColor = ConsoleColor.Red;

Les étapes à suivre sont :

  • L'ordinateur choisi un Personnage (qu'il stocke dans une variable attacker) aléatoirement dans sa liste de personnages (_computer)

  • Puis écrit dans la console la phrase
    "Computer has choosen " + attacker + " from his soldiers..."

  • L'ordinateur choisit un Personnage (qu'il stocke dans une variable target) aléatoirement dans la liste de personnages du joueur

  • Puis écrit dans la console la phrase
    "...and is going to attack your " + target

  • Enfin, appelle la fonction Attack (cf Intermediate), avec en paramètre le joueur !
    (Reregarder la doc de la fonction Attack si ce n'est pas clair)

  • Ne pas oublier de remettre la couleur de la console en blanc (car l'ordinateur a terminé de jouer)
    Pour ce faire utiliser : Console.ForegroundColor = ConsoleColor.White;

De la même manière que l'ordinateur, nous allons permettre au joueur de choisir un de ses personnages pour attaquer !
